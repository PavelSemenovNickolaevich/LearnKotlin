Задача Блюдо
Цели:

научиться создавать большое количество объектов на основе интерфейсов
применять более сложные условия вместе с коллекциями
применять коллекцию ArrayList (список объектов) с различными условиями
применять циклы по коллекциям (перебор объектов)
Сущности:

1) Ингредиент (Ingredient)
– свойство “taste”: вкус (String)
– свойство “weight”: вес (int)
– свойство “name”: название (String)

2) Блюдо (Dish)
– метод “cook”: приготовить блюдо
– свойство “ingredients”: ингредиенты блюда (коллекция, например ArrayList)
– свойство “name”: название (String)

Под сущностью подразумевается важный смысловой объект, который участвует в задаче (интерфейс + классы-реализации, либо просто отдельный класс без интерфейса).

Условия:
Нельзя приготовить блюдо, где:

смешано солёное со сладким – выводить сообщение “нельзя смешивать солёное со сладким”
добавили больше 4х ингредиентов (меньше – можно) – сообщение “нельзя добавить больше 4-х ингредиентов”
Для вкуса можно использовать 1 из 4 значений: “sour” – кислый, “sweet” = сладкий, “salty” = солёный, “bitter” – горький.

Запуск
Можно создавать любое количество классов для ингредиентов (например, Meat, Apple, Potate и пр. – на ваше усмотрение) и добавлять их в любые блюда

В методе main нужно создать несколько блюд с любым количеством ингредиентов и вызвать методы готовки.

После успешной готовки: вывести в консоль: блюдо name готово, ингредиенты: список, общий вес: вес всех ингредиентов

Пример вывода в консоль после выполнения программы:

Блюдо 'Сэндвич' готово, ингредиенты: [хлеб бородинский, колбаса докторская, сыр голландский], общий вес: 35
Блюдо 'Суп' не может быть приготовлено! Ингредиентов не должно быть больше 4-х (у вас 5)
Блюдо 'Борщ' не может быть приготовлено! Смешано соленое со сладким.
