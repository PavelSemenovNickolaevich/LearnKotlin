Текст задачи:
Создать плейлист, добавить в него песни и проиграть их.

Сущности:

1) Плейлист (Playlist) – список, который будет содержать все песни (Song)
– функция “addSong” – добавить новую песню в конец списка
– функция “shuffle” – перемешать песни (готовая функция shuffle у list)
– свойство “songList” – список доступных песен – коллекция ArrayList

2) Песня (Song) – объект, который будем добавлять в коллекцию songList – ArrayList
– свойство “name”: название песни – String
– свойство “author”: ссылка на Author
– функция “play”: проиграть музыку – выводит в консоль: название песни и автора

3) Автор (Author) – можно использовать data class (без интерфейса, аналог POJO в Java) – просто содержит свойства и нет доп. функций
– свойство “name”: имя автора

Под сущностью подразумевается важный смысловой объект, который участвует в задаче (интерфейс + классы-реализации, либо просто отдельный класс без интерфейса).

Продумать, для каких объектов нужны интерфейсы, для каких нет.

Условия:
При проигрывании песни должно выводиться: “Играет песня название, автор автор“

Коллекцию можно перемешать функцией shuffle – list уже должен быть заполнен ранее

Нельзя добавлять одну и ту же песню дважды (используйте готовую функцию contains из ArrayList)

Запуск
Создаем объекты в main:

плейлист
авторов
любое количество объектов Song и добавляем их в плейлист
проиграть все песни из плейлиста (с помощью цикла)
перемешать песни и заново проиграть их
